# 评分拖条与填充气泡交互规范（可复用 Playbook）

> **目的**：把 MESA 面试评价 / 绩效评价公开页的「Codex 风格评分条」经验固化成可迁移模式，方便其他项目复用。  
> **本仓库落地**：`web/src/components/ScoreFillBubbles.jsx` · `web/src/pages/PublicInterviewEval.jsx`（`ScoreSlider`）· `web/src/pages/PublicPerformanceEval.jsx` · `web/src/index.css`（`score-bubble-drift`）。  
> **原则**：视觉可以炫，**落盘数据必须干净**（整数分 / 与 Excel 模板字段一致）。

---

## 1. 组件分层（推荐架构）

| 层 | 职责 | 不要做 |
|----|------|--------|
| **原生 `input[type=range]`** | 指针命中、无障碍、键盘 | 用 track 自己画填充色（透明 track，自定义 fill） |
| **自定义 Fill 条** | brand 渐变宽度、气泡宿主 | 用 `width: value%` 直接对齐拇指（见 §3） |
| **分数标签 / 光晕** | 跟拇指中心走 | 用与 fill 不同的定位公式 |
| **`ScoreFillBubbles`** | 填充区内右→左白泡 | 门控「仅 dragging」当唯一显隐（见 §5） |
| **`onChange` vs `onScrub`** | 整数落盘 vs 浮点实时 UI | 把 float 写进草稿 API / Excel |

```text
┌─ Slider root ─────────────────────────────┐
│  分数（绝对定位 @ thumbCenter）            │
│  ┌─ track (灰底 overflow-hidden) ───────┐ │
│  │  ┌─ fill (渐变, width=thumbCenter) ┐ │ │
│  │  │  ScoreFillBubbles (inset-0)      │ │ │
│  │  └──────────────────────────────────┘ │ │
│  │  光晕（可选，dragging @ thumbCenter）   │ │
│  │  <input type=range transparent />     │ │
│  └───────────────────────────────────────┘ │
└────────────────────────────────────────────┘
```

## 2. 交互基线（产品约定）

1. **离散业务分 + 连续拖动手感**：UI 用细步进（如 `step={0.01}`）拖动；提交 / 草稿 / 导出仍是整数（面试 1–10，绩效 1–100）。
2. **分数贴在白色拇指正上方**，不要固定左上角；旁路 `/10`、`/100` 文案可省略（范围已在章节标题说明）。
3. **不要用 1–10 圆点按钮行做主交互**（手机易挤、手感差）；刻度数字条也可去掉，避免与拇指分数抢注意力。
4. **有填充即可持续气泡**（中段也要有）；拖动仅作「增强」，不作唯一开关。
5. **`prefers-reduced-motion: reduce`** 时不渲染持续气泡（及其它无限循环动效）。

## 3. 核心公式：拇指内缩对齐（必读）

### 3.1 为什么会「进度条冒出圆点 / 光圈偏右」

浏览器对 `range` 拇指的水平位置是：

\[
\text{thumbCenter} = r \cdot (W_{\text{track}} - W_{\text{thumb}}) + \frac{W_{\text{thumb}}}{2}
\]

其中 \( r = \dfrac{\text{value} - \min}{\max - \min} \)（例如 1–10 时 \( r = (\text{value}-1)/9 \)）。

**错误写法**（会偏）：

- Fill：`width: ${(value/max)*100}%` 或 `width: ${r*100}%`（按满宽比例，忽略拇指直径）
- Glow / 分数：`left: ${r*100}%` + `translateX(-50%)`（同上）

拇指在中间时，错误定位会落在拇指**右侧**，fill 也会「穿出」白圆。

### 3.2 正确写法（CSS）

拇指直径与样式一致（例：`w-4` → `16px`）：

```js
const THUMB = 16;
const r = value == null ? 0 : (value - min) / (max - min);
const thumbCenter = `calc(${r} * (100% - ${THUMB}px) + ${THUMB / 2}px)`;
// fill 宽度接到拇指中心（半个拇指压在填充上，视觉正确）
const fillWidth = value == null ? "0px" : thumbCenter;
```

**凡跟拇指对齐的层（fill / 分数 / 光晕 / tooltip）必须共用同一 `thumbCenter`。**

### 3.3 未评分态

- 展示「—」、fill 宽 `0`；`input` 可暂用 `min` 作 controlled value，但不要把未评分写成已选 `min` 分进表单。
- 气泡：`fillRatio = 0` → 不渲染。

## 4. 丝滑拖动：`scrub` 浮点 + 整数 `onChange`

### 4.1 问题

若 `step={1}` 且 `value` 始终是整数，拇指只有 N 档，手感「一格一格跳」。  
若拖动时 `onChange(Math.round(x))` 且父级用整数算加权总分，LiquidLoader 会出现 **7.5 → 9** 这类跳档，小数不丝滑。

### 4.2 模式

```text
pointerdown → dragging=true，scrub=当前分
input onChange → scrub=float；onScrub(float)；仅当 round(float)≠score 时 onChange(int)
pointerup → onChange(round(scrub))；scrub=null；onScrub(null)
```

- **Form / PATCH 草稿 / Excel**：只认整数。
- **liveTotal / 加权展示 / LiquidLoader**：拖动中用 `scrubByKey[dim] ?? score`。
- LiquidLoader：拖动时可加 `instant`（关掉液面 height 过渡），避免「数字已变、液面还在缓动」的割裂感。

### 4.3 伪代码

```js
function handleChange(raw) {
  const f = clamp(Number(raw), min, max);
  setScrub(f);
  onScrub?.(f);                 // 仅 UI 汇总
  const rounded = Math.round(f);
  if (rounded !== score) onChange(rounded); // 落盘整数
}
```

## 5. Codex 填充气泡（`ScoreFillBubbles`）

### 5.1 宿主

气泡必须挂在 **fill 容器内部**，且 fill 设 `overflow-hidden` + `relative`，这样泡只出现在已填充渐变里，并以**拇指（填充右缘）**为起点向左漂。

### 5.2 显隐门控（踩过的坑）

| 做法 | 结果 |
|------|------|
| `show = dragging && fillRatio >= 0.45` | 同页多条满分时，**只有正在拖的那条有泡**；松手全灭 |
| `show = fillRatio >= MIN_FILL`（如 0.05） | 中段 / 满分均可持续；`dragging` 只增强强度 ✅ |

### 5.3 动画注意：`left` vs `transform`

要用 **相对 fill 容器的 `left` 百分比** 做右→左位移。  
若只用 `transform: translateX(-N%)`，百分比相对**自身宽度**，几乎看不见「穿越轨道」的位移。

推荐 keyframes 形态：

```css
@keyframes score-bubble-drift {
  0%   { left: var(--bubble-start); opacity: 0; transform: translateY(0) scale(0.85); }
  12%  { opacity: var(--bubble-opacity); }
  50%  { transform: translateY(-2px) scale(1); }
  100% { left: var(--bubble-end); opacity: 0; transform: translateY(2px) scale(0.7); }
}
```

多泡：错开 `size / top / start / end / delay / duration / opacity`；近满分或 `active`（拖动）时可加密度与不透明度。

### 5.4 `fillRatio` 语义

- 传给气泡的是 **0–1 的填充比例**（与 §3 的 \(r\) 一致），不是业务分本身。
- 极窄填充时可 `Math.max(r, 0.08)` 略抬，避免泡挤成一团；仍建议 `MIN_FILL`（如 0.05）以下直接不渲染。

## 6. 样式清单（可直接抄）

- Track：`h-2.5 rounded-full bg-neutral-200 overflow-hidden`
- Fill：`h-full rounded-full` + 品牌渐变 + `overflow-hidden`
- Thumb：白底 + brand 描边 + 轻阴影；`active:scale-125` 反馈拖动
- Range input：`appearance-none bg-transparent`；webkit/moz track 透明；thumb 与 `THUMB` 像素一致
- 拖动中关掉 fill 的 `width` transition，松手再开短过渡（~120ms）
- 分数用 `tabular-nums`，整数变档时可用短 GSAP scale bounce（非整段 float 每帧）

项目内动画栈：**GSAP + CSS keyframes**；不要为滑条再引 framer-motion。

---

## 7. 无障碍与可访问性

- `aria-valuemin` / `aria-valuemax` / `aria-valuenow` / `aria-valuetext`（未评分用「未评分」）
- 标签用整数业务分，不要把 float scrub 读给读屏（或 valuetext 仍报 round）
- 气泡 `aria-hidden`；减少动效偏好下整块不挂载
