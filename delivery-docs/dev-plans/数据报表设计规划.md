# MESA Recruit · 数据报表模块设计规划文案(终版 · 含 GSAP 动画方案)

> 路径:`/reports`
> 范围:页面信息架构 + 业务功能 + 数据口径 + 二期增强 + 全局规范 + **GSAP 动画落地方案(含代码)**
> 状态:可直接交付设计与开发

---

## 第一部分 · 模块定位与目标用户

数据报表 `/reports` 是招聘工作台的核心数据洞察入口,服务三类角色:招聘负责人(看大盘 + 看趋势)、HRBP(看部门健康度)、用人部门 Leader(看自己 JD 的进展)。设计目标是让用户在**一屏之内**看清当前漏斗状态、上一周期对比、以及问题最严重的 JD/部门定位,再通过下钻到候选人列表完成行动闭环。

视觉与动效目标:在保证数据可读性的前提下,通过克制、统一的 GSAP 动画,让"数据被看见的瞬间"有节奏、有焦点、有呼吸感,而不是一次性 dump 在屏幕上。

## 第二部分 · 一期核心(MVP)

### 1. 顶部全局筛选区

固定吸顶,四组控件:

- **时间维度切换** — 快捷 Tab(`今日 / 本周 / 本月 / 本季度 / 本年`)+ `自定义时间段`(日期范围选择器,最长 3 年)。默认"本月"。所选维度自动驱动:对比基准(日/周/月/季/年)、趋势图粒度、导出文件命名。
- **JD 筛选** — 多选下拉 + 搜索,支持"全选 / 反选 / 清空"。
- **部门筛选** — 树形多选,父子级联,支持"仅看我负责的部门"快捷过滤。
- **右侧动作区** — `刷新`、`导出(Excel / CSV)`、`订阅周报 / 月报邮件`。

筛选条件持久化到 URL Query,便于分享与刷新保留状态。

### 2. 核心指标卡(KPI Cards)

5 张卡片,统一三段式结构:**主数值 + 环比 + 迷你 sparkline**。

| 卡片         | 口径                                           |
| ------------ | ---------------------------------------------- |
| 新增简历数   | 当期上传到系统的简历份数(按上传时间)           |
| 候选人总量   | 当期实际进入流程的候选人,去重,已删除候选人扣减 |
| 在招岗位数   | 当期状态为"招聘中"的 JD 数(取期末快照)         |
| 进入面试人数 | 当期至少触发一次面试安排的候选人,去重          |
| 成功入职人数 | 当期状态流转至"已入职"的候选人                 |

环比文案:`较上一{周期} ▲ 12.5%`,绿涨红跌,缺失展示 `—`。点击卡片下钻对应列表。

### 3. 招聘漏斗阶段分布

垂直漏斗:`待筛选 → 已沟通 → 已安排面试 → 面试通过 → 已发 Offer → 待入职 → 已入职`,旁路 `已淘汰 / 已放弃`。每段同屏展示:当期人数、占上一阶段的转化率、较对比期变化。点击阶段条 → 右侧抽屉打开该阶段候选人列表。

### 4. 趋势图

主折线图,粒度自适应时间维度(今日→小时;本周→天;本月→天可切周;本季→周可切月;本年→月可切季;自定义→系统推荐 + 手动覆盖)。顶部指标 chip 多选,最多叠加 3 条曲线;"对比模式"开关 → 浅色虚线叠加上一周期同指标;Hover Tooltip 同时展示绝对值与环比差。

### 5. JD 维度分析

表格列:`JD 名称 / 部门 / 候选人总数 / 各阶段 mini bar / 平均处理时长 / 较上期变化`。点击 JD 行 → 候选人列表抽屉。多选 JD 时下方追加堆叠柱状图,横向对比阶段构成。

### 6. 部门维度分析

部门卡片矩阵 + 部门对比柱状图。卡片点击下钻"该部门所有 JD + 候选人"二级列表,可再按 JD 二次拆分。

### 7. 下钻列表统一规范

右侧抽屉占屏 60%,字段标准化:`姓名 / 联系方式(脱敏) / 关联 JD / 所属部门 / 当前阶段 / 进入阶段时间 / 最近操作人 / 操作`。顶部面包屑显示当前下钻路径,提供"导出当前列表"按钮。已删除候选人默认隐藏,管理员可开"显示已删除"用于审计,但不计入任何统计。

### 8. 数据口径

候选人计数以创建时间为准;阶段人数以"进入该阶段时间戳"为准;同一候选人区间内多次流转时,趋势图按事件计数、KPI 按期末状态去重。软删除 `is_deleted=true` 在所有聚合 SQL 层统一过滤。环比基准:今日 vs 昨日 / 本周 vs 上自然周 / 本月 vs 上自然月 / 本季度 vs 上自然季度 / 本年 vs 去年同期 / 自定义 vs 向前推等长区间。

## 第三部分 · 二期增强(全部拆分为独立板块)

每个板块按 `目标 / 字段 / 交互 / 口径 / 优先级 / 工期` 五件套呈现。

### 二期-1 · 招聘渠道来源分析
**目标**:量化每个简历来源渠道的获取效率与转化质量。
**字段**:渠道、新增简历、进入面试、Offer、入职、各阶段转化率、人均成本(可选)。
**交互**:独立 Tab「渠道分析」,渠道对比表 + 渠道转化漏斗 + A/B 对比。
**口径**:简历创建时强制绑定 `source_channel` 枚举,历史数据走"渠道补录"后台。
**优先级**:P1 / **工期**:2.5 人周。

### 二期-2 · 招聘官 / HR 个人绩效看板

> **命名免责**:此处「HR 绩效」= 招聘官工作量/漏斗贡献指标,属于 `/reports` 报表能力。
> **≠** 产品页 `/performance` 的**员工绩效评价**(PerformanceEvaluation · 自评/主管打分表)。员工绩效评价见 `delivery-docs/src/06_performance_evaluation.md`。
**目标**:量化每位 HR 的工作量与产出质量。
**字段**:负责 JD 数、新增候选人、推进面试、Offer、入职、平均响应时长、平均阶段推进时长。
**交互**:Tab「HR 绩效」,表格 + 排行榜双视图,点击 HR 下钻其 JD 与候选人,支持"周报快照"导出。
**权限**:HR 看自己 / Leader 看部门 / Admin 看全公司。
**优先级**:P1 / **工期**:2 人周。

### 二期-3 · Offer–入职周期与流失分析
**目标**:发现"发了 Offer 但没入职"的高危环节。
**字段**:Offer 发放/预计/实际入职时间、平均周期、流失数、流失原因分布。
**交互**:页面下方「Offer 健康度」卡片组 + 流失原因饼图,点击扇区下钻候选人。
**口径**:状态机新增 `offer_declined` 终态,标记流失强制选原因枚举 + 备注。
**优先级**:P1 / **工期**:1.5 人周。

### 二期-4 · 各阶段流失原因分布
**目标**:定位漏斗最大渗漏点。
**字段**:阶段、淘汰人数、淘汰原因、原因占比、环比变化。
**交互**:漏斗图右侧并列瀑布图,Hover 漏斗阶段联动瀑布图高亮。
**口径**:每次"淘汰/放弃"弹窗强制选原因枚举,历史数据迁移给 `legacy_unknown`。
**优先级**:P2 / **工期**:2 人周。

### 二期-5 · 面试官打分分布与一致性分析
**目标**:衡量面试官打分倾向、一致性、与最终入职质量相关性。
**字段**:面试场数、平均打分、标准差、推荐通过率、被推荐者入职率、面试时长均值。
**交互**:Tab「面试官分析」,箱型图 + 推荐通过率排行 + 一致性矩阵。
**口径**:依赖结构化面试评分卡。
**优先级**:P2 / **工期**:3 人周。

### 二期-6 · 自定义指标看板(拖拽组合)
**目标**:让高级用户自由组合"维度 × 指标 × 图表",保存为个人看板。
**字段(配置层)**:看板名、组件列表、各组件(数据源、维度、指标、过滤、图表、布局)。
**交互**:左侧组件库 + 中部 12 列栅格画布 + 右侧配置面板;支持保存、另存为、设为默认、分享只读链接。
**口径**:基于已有"原子指标"组合,新增原子指标走开发流程。
**优先级**:P3 / **工期**:6–8 人周。

### 二期-7 · 自动化订阅报表(邮件 / 企微 / 钉钉)
**目标**:让管理者无需打开系统就能定期收到关键指标。
**字段**:频率、时间、收件人、指标快照、推送渠道。
**交互**:顶部"订阅"弹窗 + 个人中心"我的订阅"管理页;推送图文 HTML + PDF 附件 + 回跳链接。
**优先级**:P2 / **工期**:2 人周。

### 二期-8 · 目标达成率
**目标**:支持设置季度/年度招聘目标,实时展示达成进度。
**字段**:目标周期、目标值、实际值、达成率、剩余天数、预期达成线。
**交互**:顶部"目标达成"横向进度条 + KPI 卡叠加"距目标 ±X 人"。
**口径**:Admin 在「目标管理」后台设置全局/部门/JD 三级,优先级 JD > 部门 > 全局。
**优先级**:P2 / **工期**:2 人周。

### 二期-9 · 异常预警与自动洞察
**目标**:系统主动发现异常并提示。
**示例规则**:某 JD 30 天无新增 → 冷启动告警;某 JD 面试通过率 < 全公司均值 50% → 面试要求过高;某部门入职环比 -40% → 推送高管。
**交互**:报表顶部"洞察 Banner"轮播 Top 3 + 独立「洞察中心」页。
**优先级**:P3 / **工期**:4 人周。

## 第四部分 · 全局通用规范

### 1. 权限模型
`Admin`(全公司)/ `招聘 Leader`(本部门 + 下属 HR)/ `HR`(自己负责的 JD 与候选人,可看部门聚合)/ `用人部门 Leader`(本部门 JD,不可见 HR 个人绩效)。后端聚合接口校验,前端按角色隐藏 Tab。

### 2. 性能
首屏 ≤ 1.5s(p95);筛选切换 ≤ 800ms(p95);趋势图 > 365 点降采样;Excel > 10 万行走异步任务邮件投递。聚合落 OLAP / 物化视图,小时级刷新;高实时性 KPI 走主库。

### 3. 空状态与异常态
全无数据 → 居中插画 + 引导文案 + CTA;模块无数据 → 模块内空状态,不影响其他模块;接口异常 → 模块级降级"重试";权限不足 → 灰显 + 文案。

### 4. 移动端
桌面优先,移动精简版:顶部 KPI + 漏斗 + 一张主趋势图,复杂控件折叠为底部抽屉,横屏切桌面布局。

### 5. 国际化
指标名、阶段名走 i18n key;时区按用户所在时区聚合(UTC 落库,展示时转换);日期格式跟随浏览器 locale。

### 6. 埋点
事件命名 `reports_<action>_<target>`;后端聚合接口打 trace,慢查询 > 1s 告警。

### 7. 可访问性
颜色对比度 ≥ WCAG AA;图表提供"查看数据表格"切换;关键操作可键盘操作;**所有动画必须遵守 `prefers-reduced-motion`**(详见第五部分)。

## 第五部分 · GSAP 动画设计规范(本次新增重点)

### 1. 设计原则

1. **服务数据,不喧宾夺主** — 动画用于"引导视线 + 解释关系",不做炫技;单次动画时长 0.3–0.8s,缓动统一 `power2.out` 或 `expo.out`。
2. **节奏统一** — 全局采用统一节拍变量:`--anim-fast: 0.25s; --anim-base: 0.5s; --anim-slow: 0.8s`,代码中映射为 GSAP duration。
3. **错峰入场(Stagger)** — 同类元素入场用 `stagger: 0.06`(卡片)/`0.04`(列表),避免一次性闪现。
4. **可中断** — 任何动画在用户切换筛选、关闭抽屉时必须可被打断;统一在组件 unmount 时 `ctx.revert()`。
5. **降级** — `@media (prefers-reduced-motion: reduce)` 下,所有 GSAP 动画 duration 归零,只保留 opacity 切换。
6. **依赖**:`gsap@^3.12`、`@gsap/react`(useGSAP Hook)、按需引入 `ScrollTrigger`、`Flip`、`Observer`。

### 2. 公共初始化(`src/anim/gsap.ts`)

```ts
import { gsap } from 'gsap';
import { ScrollTrigger } from 'gsap/ScrollTrigger';
import { Flip } from 'gsap/Flip';

gsap.registerPlugin(ScrollTrigger, Flip);

// 统一节拍
export const D = { fast: 0.25, base: 0.5, slow: 0.8 };
export const E = { out: 'power2.out', expo: 'expo.out', back: 'back.out(1.4)' };

// 全局 reduce-motion 适配
const mql = window.matchMedia('(prefers-reduced-motion: reduce)');
if (mql.matches) gsap.globalTimeline.timeScale(0.001);
mql.addEventListener('change', e => gsap.globalTimeline.timeScale(e.matches ? 0.001 : 1));
```

### 3. 模块级动画方案

#### 3.1 顶部筛选区(吸顶 + 切换反馈)

**场景**:页面加载时筛选条由上方滑入;切换时间 Tab 时下方 KPI 与图表需有"刷新感"。

```tsx
import { useGSAP } from '@gsap/react';
import { gsap } from 'gsap';
import { D, E } from '@/anim/gsap';

export const FilterBar = () => {
  const ref = useRef<HTMLDivElement>(null);

  useGSAP(() => {
    gsap.from(ref.current, { y: -24, opacity: 0, duration: D.base, ease: E.out });
  }, { scope: ref });

  const onRangeChange = () => {
    // 时间维度切换 → 通知下方区域抖动重渲染
    gsap.fromTo('[data-anim="dash-zone"]',
      { opacity: 0.4, y: 8 },
      { opacity: 1, y: 0, duration: D.fast, ease: E.out, stagger: 0.04 });
  };

  return <div ref={ref} data-anim="filter-bar">...</div>;
};
```

#### 3.2 KPI 卡片(入场 + 数字滚动 + 环比颜色脉冲)

**场景**:卡片错峰入场;主数值用 GSAP 做 `0 → 目标值` 数字滚动;环比上涨时绿色脉冲一次。

```tsx
useGSAP(() => {
  const tl = gsap.timeline();
  tl.from('.kpi-card', {
    y: 24, opacity: 0, duration: D.base, ease: E.out, stagger: 0.08,
  });

  // 数字滚动
  document.querySelectorAll<HTMLElement>('.kpi-number').forEach(el => {
    const target = Number(el.dataset.value || 0);
    const obj = { v: 0 };
    gsap.to(obj, {
      v: target, duration: D.slow, ease: E.expo,
      onUpdate: () => { el.textContent = Math.round(obj.v).toLocaleString(); },
    });
  });

  // 涨跌脉冲
  gsap.fromTo('.kpi-delta.up',
    { backgroundColor: 'rgba(34,197,94,0.25)' },
    { backgroundColor: 'rgba(34,197,94,0)', duration: 1.2, ease: 'power1.out' });
}, []);
```

#### 3.3 招聘漏斗(自上而下展开 + 阶段条宽度过渡)

**场景**:漏斗每一阶段宽度从 0% 缓动至真实占比,自上而下错峰展开。

```tsx
useGSAP(() => {
  gsap.from('.funnel-stage', {
    opacity: 0, x: -16, duration: D.base, ease: E.out, stagger: 0.06,
  });

  document.querySelectorAll<HTMLElement>('.funnel-bar').forEach(bar => {
    const w = bar.dataset.width || '0%';
    gsap.fromTo(bar, { width: '0%' }, { width: w, duration: D.slow, ease: E.expo });
  });
}, []);
```

阶段条 Hover 时高亮其他阶段半透明:

```ts
const onEnter = (i: number) => gsap.to('.funnel-stage', {
  opacity: (idx) => idx === i ? 1 : 0.35, duration: D.fast,
});
const onLeave = () => gsap.to('.funnel-stage', { opacity: 1, duration: D.fast });
```

#### 3.4 趋势图(折线 path 绘制 + Tooltip 弹入)

**场景**:折线用 `strokeDasharray` 描边动画,从左到右"画"出来;Tooltip 弹出用轻微 `back.out`。

```ts
const path = svg.querySelector<SVGPathElement>('.trend-line')!;
const len = path.getTotalLength();
gsap.set(path, { strokeDasharray: len, strokeDashoffset: len });
gsap.to(path, { strokeDashoffset: 0, duration: D.slow, ease: E.expo });

// 数据点依次显现
gsap.from('.trend-dot', {
  scale: 0, transformOrigin: '50% 50%',
  duration: D.fast, ease: E.back, stagger: 0.03, delay: 0.3,
});
```

切换粒度(天/周/月)时使用 GSAP **Flip** 平滑过渡坐标轴标签:

```ts
import { Flip } from 'gsap/Flip';

const state = Flip.getState('.x-axis-label');
// ...更新 DOM
Flip.from(state, { duration: D.base, ease: E.out, absolute: true, stagger: 0.02 });
```

#### 3.5 JD / 部门表格(行入场 + 排序换位)

**场景**:表格行入场用 stagger;用户切换排序时,行的位置变化用 Flip 平滑换位,而不是瞬切。

```tsx
useGSAP(() => {
  gsap.from('.jd-row', { opacity: 0, y: 12, duration: D.base, ease: E.out, stagger: 0.04 });
}, []);

const onSort = (sorter: SortFn) => {
  const state = Flip.getState('.jd-row', { props: 'opacity' });
  applySort(sorter);                       // 重新渲染列表
  Flip.from(state, {
    duration: D.base, ease: E.out, absolute: true, stagger: 0.02,
    onEnter: (els) => gsap.from(els, { opacity: 0, y: 8 }),
    onLeave: (els) => gsap.to(els, { opacity: 0, y: -8 }),
  });
};
```

#### 3.6 下钻抽屉(滑入 + 内容错峰)

```ts
const openDrawer = () => {
  const tl = gsap.timeline();
  tl.fromTo('.drawer-mask', { autoAlpha: 0 }, { autoAlpha: 1, duration: D.fast })
    .fromTo('.drawer-panel',
      { xPercent: 100 },
      { xPercent: 0, duration: D.base, ease: E.out }, '<')
    .from('.drawer-row',
      { opacity: 0, x: 16, duration: D.fast, ease: E.out, stagger: 0.03 }, '-=0.1');
};

const closeDrawer = () => {
  gsap.to('.drawer-panel', { xPercent: 100, duration: D.fast, ease: 'power2.in' });
  gsap.to('.drawer-mask', { autoAlpha: 0, duration: D.fast });
};
```

#### 3.7 ScrollTrigger:JD / 部门区进入视口才动

```ts
import { ScrollTrigger } from 'gsap/ScrollTrigger';

gsap.from('.dept-card', {
  opacity: 0, y: 32, stagger: 0.08, duration: D.base, ease: E.out,
  scrollTrigger: { trigger: '.dept-zone', start: 'top 80%', once: true },
});
```

#### 3.8 异常态 / 空状态(微动效)

空状态插画呼吸缩放,提示用户"页面正常但暂无数据":

```ts
gsap.to('.empty-illustration', {
  scale: 1.04, duration: 1.6, ease: 'sine.inOut', yoyo: true, repeat: -1,
});
```

加载骨架屏闪光:

```ts
gsap.to('.skeleton', {
  backgroundPosition: '200% 0', duration: 1.2, ease: 'none', repeat: -1,
});
```

### 4. React 集成约定

- 统一使用 `@gsap/react` 提供的 `useGSAP` Hook,自动处理 `gsap.context()` + 清理。
- 所有动画作用域内必须传 `scope: ref`,避免选择器跨组件污染。
- 列表场景与 SSR / Suspense 共存时,确保 DOM 挂载后再触发动画(在 `useLayoutEffect` 等价时机执行)。

### 5. 性能与可访问性约束

- 优先动画 `transform / opacity`,避免动画 `width / top / left`(漏斗条宽度等不可避免时,确保只动一次)。
- 不在同一帧并行启动 > 30 个 tween,长列表用 stagger + `lazy: true`。
- 单帧脚本耗时控制在 ≤ 8ms,大数据表禁用入场动画(超过 100 行直接跳过 stagger)。
- `prefers-reduced-motion: reduce` 一律让动画退化为瞬时切换。
- 不在动画期间禁用交互;关键 CTA 始终可点击。

### 6. QA 自检清单(上线前)

1. 切换时间 Tab、JD/部门筛选,所有图表是否平滑刷新且无闪烁。
2. 数字滚动结束后展示值与接口返回值一致(浮点四舍五入正确)。
3. 抽屉打开/关闭过程中,快速重复点击不会出现错位或残影。
4. 表格排序使用 Flip 后,行高度与对齐保持稳定。
5. `prefers-reduced-motion: reduce` 时,所有动画静默。
6. 滚动到 JD/部门区域后,动画只触发一次,后续不重复。
7. 抽屉、Toast、Modal 关闭时,所有 GSAP tween 被正确 kill,无内存泄露。
8. 移动端动画时长缩短为桌面的 0.75x(在 `D` 中按 viewport 自适应)。
